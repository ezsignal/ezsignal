# Security Policy

## Supported Versions

The latest version of EZ SIGNAL HQ is actively maintained and receives security updates.

## Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public GitHub issue.

Instead:

1. Contact the maintainers privately.
2. Provide detailed reproduction steps.
3. Allow reasonable time for investigation and remediation.

We will acknowledge reports as quickly as possible and work toward a responsible disclosure process.

## Security Practices

EZ SIGNAL HQ includes:

* Protected production endpoints
* Bearer token authentication
* Cron secret validation
* Operational audit workflows
* Deployment verification procedures

Security improvements and responsible disclosure reports are welcome.
